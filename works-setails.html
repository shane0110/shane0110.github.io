雅茗 - 電商網站

2024 - now

我設計的雅茗電商 LOGO，使用了 Adobe Illustrator 製作，並選擇黃金色作為主要色調，這讓整體設計感覺既高雅又有質感。黃金色通常象徵著奢華與成功，非常適合電商品牌來傳達一種值得信賴和高品質的形象。這樣的設計也能幫助品牌在視覺上與競爭對手區分開來，讓消費者更容易記住這個 LOGO。

個人在專案的定位

在雅茗電商的專案中，我負責的不只是設計，還承擔了裡面課程功能的開發。因為我是團隊裡唯一有設計背景的人，大家經常會把自己的介面設計拿來找我討論，看看有沒有什麼可以優化的地方。這種情況讓我有機會從設計和開發的角度，提供更全面的建議。 在處理這些討論時，我也不只是單純考慮美觀，而是會把使用者體驗和功能性放在一起考量，確保整體操作順暢。對我來說，這樣的角色不僅讓我能夠運用設計能力，還強化了我在前端開發中的實踐經驗。這個過程雖然有挑戰，但也讓我更清楚怎麼讓設計和功能無縫結合，最終呈現出一個統一且實用的產品。

  • 在專案中，我也會與隊友討論排版對功能的影響，並特別注意 RWD 細節，確保在各裝置上都有一致的使用體驗。
  • 在雅茗電商的專案中，除了設計和開發課程功能，我也同時考慮了 RWD（響應式網頁設計）的細節。由於專案需要在不同裝置上呈現一致的體驗，我在設計介面時特別注意到如何讓每個元件在桌機、平板和手機上都能流暢地顯示。 RWD 的部分挑戰在於確保頁面佈局靈活且具適應性，尤其是在調整圖片尺寸、文字排版以及按鈕的可點擊範圍時，必須平衡美觀和實用性。我也不斷測試每一個設計，確保它在不同的螢幕尺寸下依然保持一致的品牌風格和良好的使用體驗。這樣的細節工作，讓整個專案在使用者體驗上更加完善。
  • 這讓我想到在專案中與其他人互相討論的過程中，如何同時保持進度順利推進，確保每個人都朝同樣的目標前進。

在這個專案中，我學到了如何有效地與他人溝通，並學會了在團隊合作中如何將設計和功能需求整合在一起，確保專案順利進行。

個人在專案的功能說明

在技術上，我使用了多種技術來實現這個專案的功能，主要包括 HTML、CSS、JavaScript、React、SQL 和 Node.js。首頁部分，我運用了 React 的組件來增強使用者互動體驗，實現了一些動態效果。例如，我們加入了茶品推薦的滑動效果，讓訪客能更直觀地瀏覽各種茶品。 在課程部分，我實作了篩選功能，讓用戶可以根據不同的需求（如茶類型、價格範圍等）來篩選課程，提供更個性化的學習體驗。此外，我也負責課程部分的 CRUD 功能，使得使用者能夠方便地新增、修改或刪除自己的課程資料，確保平台的靈活性和實用性。 整體來說，這個專案不僅讓我在技術上得到了提升，也讓我更加理解如何將設計理念與實際開發結合，創造出一個能夠吸引顧客的電商平台。這次經驗對我未來在前端開發的道路上是非常寶貴的！

  • 在專案中，我負責了首頁的切版工作，並實現了一些功能。這讓我能夠運用設計和開發的技巧，將介面從設計轉化為實際的網頁，確保整體風格一致且功能正常運作。
  • 因為我們的電商網站有前後台系統，我負責將商品部分通過 API 進行串接，讓前端能夠即時獲取後端的商品資料。在這個過程中，我使用了 React 的載入動畫（animation），為商品顯示加入了滑動效果，提升了使用者體驗。這樣的動畫不僅讓頁面看起來更生動，還讓使用者在等待商品資料加載時，感覺過程更順暢，進一步提高了網站的互動性與專業感。
  • 在課程部分，我同樣使用了其他 React 載入動畫來進行製作，並且加入了滑動效果。這樣的設計不僅讓課程列表看起來更具吸引力，還能夠增強使用者在瀏覽時的互動體驗。透過這些動畫效果，我希望能讓使用者在切換課程時感受到更流暢的過程，進一步提升整體的使用者滿意度。
  • 在這個專案中，我製作了一個有互動效果的台灣地圖。這個地圖不僅美觀，還能讓使用者輕鬆瀏覽不同地區的課程或商品資訊。透過互動設計，使用者可以點擊地圖上的不同區域，快速獲取相關內容，提升了整體的使用體驗和互動性。這樣的設計不僅增加了網站的趣味性，還幫助使用者更好地理解我們的產品與服務。
  • 接下來是課程的部分，這是我主要負責的功能區塊，也串後台的資料庫大概40多筆的資料，同時也有寫篩選的功能，可以依照我需要的課程地區價格去做篩選。
  • API用Fetch配合迴圈方式將他寫到前台使用 forEach 迴圈遍歷獲取到的數據，並為每一個項目創建一個新的 HTML 元素來顯示資料。
  • 點擊進去後，一樣是用API用Fetch配合迴圈方式將他寫到介面，這邊寫入三張照片同時也使用react的載件讓他有滑動效果
  • 底下的其他課程有寫入隨機，能讓他隨機生成他料庫的資料
  • 這是課程的後台介面，用於使用crud的功能。
